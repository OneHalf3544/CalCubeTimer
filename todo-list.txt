Процесс собрки
* Запускается таймер после дисквалификации по времени инспекции (нужно хотя бы начинать инспекцию заново для новой сборки)
* Почуму-то дважды генерируется скрамбл на запуске. (Не афектит пользовательские сценарии. Просто незапланированные вызовы)
* Генерация скрамблов после исчерпания импортированных
* Повтор скрамблов не учитывает добавляемые в конец списка (нужно зацикливание)

Отображение данных
* При повторных попытках скорки, подсвечивать текущий скрамбл в списке (импортированные списки отображать как DNS?)
* Надо переделать переиспользование старых скрамблов. Заменить выбор по индексу на какой-нибудь повтор по правой кнопке в списке прошлых решений
* Не работает полноэкранный режим таймера во время сборки (похоже, что чинится перезапуском)
* Цвет текста "лучшее RA"
* Неправильно выделяется цветом rollingAverage в таблице
* Не работает отображение скрамбла для megaminx
* Остается красный шрифт при основном времени сборки после инспекции
* Обшая статистика под скрамблом не зависит от текущего типа головоломки. (для трешки отборажаются рекорды от 2x2x2)
* Нет обображения выделения в таблице сессий
* Столбцы таблицы настройки скрамблов не подгоняются под размер окна

Редактирование данных
* Редактирование скрамбла после сборки
* Подгружать данные сессий только по необходимсти, а не держать все в памяти
* Не меняется тип головоломки при ее смене (только после явного создания новой сессии)
* Мержить сессии по запросу
* Отдельные кнопки для "добавить время/пользователя/тег/.."

Сборка дистрибутива
* Не открывается about -> documentation
* :distZip создает и мега-jar, содержащий все, и добавляет зависимости отдельными файлами
* Определиться с папкой логов в production-версии
